# Architecture Decision Records (ADRs)

This directory stores immutable records of major technical and architectural decisions.

## How to Write an ADR
Before deciding on an architecture, run the architecture review workflow (if available in `.claude/workflows/`).
ADRs should capture:
1.  **Title:** Short noun phrase.
2.  **Context:** The problem, constraints, and alternative options considered.
3.  **Decision:** What we are doing.
4.  **Consequences:** Trade-offs and impact on other domains.
